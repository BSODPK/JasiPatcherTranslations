 Il Patcher presenta otto tipi di Patch.

 1. Patch al Codice (Xposed non richiesto) :- This Patch Code Of The App,When You Update The App Then Before Updating 'Unpatch' The App Otherwise You Will Get Not Installed Error (Only Required On Kitkat And Below) After Installing You Have To Patch It Again. (Need To Patch Again ie. On Updates).

 2. Patch Dalvik (Xposed non richiesto) :- This Patch Cache Of The App,But You Can Update Easily No Need To Unpatch First But Reinstalling Or Updating Will Remove The Patch So You Have To Patch It Again. (Need To Patch Again ie. On Updates).

 3. Patch Licenza (Xposed richiesto) :- (Run The Application At least Once Before Patching) This Patch Google License And InApp Implementation And System Is Hooked,App Will Take Fake Data As 'LEGIT' Data Like Response From Google Servers.It Is Very Effective,Butt Hurt For App Developers. (No Need To Patch Again ie. On Updates [Only Patch One Time]).

 4. Patch Native (Xposed non richiesto) :- It Patches Native Codes,Basically For Games. (Need To Patch Again ie. On Updates).

 5. Salva Patch (Xposed non richiesto) :- (Run The Application At least Once Before Patching) It Makes Changes In Data,It Can Also Be Used Alone Or Along With Other Patches Based On App Requirements. (No Need To Patch Again ie. On Updates [Only Patch One Time]).

 6. Patch di Aggancio (Xposed richiesto) :- It Hooks Application Dynamically Makes Changes In It,Again Pain In The ASS For App Developers. (No Need To Patch Again ie. On Updates [Only Patch One Time]).

 7. Patch Moduli (Xposed non richiesto) :- Fix System Checks,Some Apps Only Need This But Sometime Hard Protected Apps Need This Along With Other Patches,It Can Also Be Used Alone Or Along With Other Patches Based On App Requirements. (Need To Patch Again ie. On Updates).

 8. Genera Patch (Xposed non richiesto) :- (Run The Application At least Once Before Patching) It Is Alternate Of License Patch But Works Without Xposed,It Uses System Default Results Rather Than Manipulating It Like In License Patch. (No Need To Patch Again ie. On Updates [Only Patch One Time]).

 NOTA :- Se un'app richiede più di una patch, solo quelle ad alta prioritá saranno mostrate nei dettagli (Prioritá :- Patch Licenza > Patch di Aggancio > Patch al Codice > Patch Dalvik > Patch Native > Patch Moduli > Salva Patch).
Le patch possono funzionare o meno quando le app vengono aggiornate, ma funzioneranno sempre per le versioni specifiche per cui sono state create.

 Se il Patcher non funziona o chrasha puoi consultare la sezione Risoluzione problemi (Patcher > Menu > Risoluzione Problemi) Segui quanto riportato. Se hai ancora problemi prendi un Log (Patcher > Menu > Prendi un Log) e inviamelo tramite email (Patcher > Menu > Info > Invia Mail).
